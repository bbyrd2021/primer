Primer MVP

Full build plan for a working prototype in 5 days. This version adds the Paper Organizer — structured per-paper extraction that turns uploaded PDFs into a searchable card library, alongside the RAG synthesis brief.

What changed from v1: We added a second core feature — structured paper card extraction. Users get both a scannable table of every paper (methodology, results, your fields from the spreadsheet) AND the narrative synthesis brief. Two outputs, one pipeline, dramatically higher perceived value.

✦ Paper Cards — Structured Extraction

This is the new feature. For every uploaded PDF, Primer makes one Claude call to extract structured fields — the same fields you've been filling in manually in your spreadsheet. The result is a JSON card stored per session.

The Extraction Prompt

This lives in core/prompts.py. The key design: ask Claude to output strict JSON only, no markdown, no preamble. Easy to parse, hard to hallucinate.

# core/prompts.py

EXTRACTION_PROMPT = """
You are a research paper analysis assistant. Extract structured information
from the provided paper text. Output ONLY valid JSON — no markdown, no preamble,
no explanation. If a field cannot be determined from the text, use null.

RESEARCH CONTEXT (use this to score relevance):
{research_question}

PAPER TEXT:
{paper_text}

Output this exact JSON schema:
{{
  "title": "Full paper title as written",
  "venue": "Conference or journal name only (e.g. CVPR, Pattern Recognition)",
  "year": 2024,
  "task": "Primary ML task (e.g. 3D object detection, semantic segmentation)",
  "modality": "Sensor modality (Camera, LiDAR, Camera+LiDAR, Radar, etc.)",
  "methodology": "1-2 sentence description of the core technical contribution",
  "results": "Key quantitative results as stated in the paper (copy exact numbers)",
  "datasets": ["list", "of", "dataset", "names", "used"],
  "pretraining": "Pretraining or weight initialization strategy, or null",
  "code_available": true/false,
  "code_url": "GitHub or project URL if mentioned, or null",
  "synthesis_note": "2-3 sentence note on how this paper relates to the research question above. Be specific about methodology relevance.",
  "relevance_score": 1-5,
  "tier": 1/2/3,
  "key_limitations": "Main limitations mentioned by authors, or null"
}}

Relevance scoring guide:
5 = Directly addresses the research question, core methodology
4 = Highly related, significant overlap with research question
3 = Relevant dataset or partial methodology overlap  
2 = Background context, loosely related
1 = Minimal relevance

Tier guide:
1 = Must-read for this research (relevance 4-5)
2 = Should-read (relevance 3)
3 = Optional background (relevance 1-2)
"""

The Extraction Module

# core/extract.py
import anthropic
import json
import os
from pathlib import Path
from core.prompts import EXTRACTION_PROMPT

client = anthropic.Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))
CARDS_DIR = Path("cards_db")
CARDS_DIR.mkdir(exist_ok=True)


def extract_paper_card(
    paper_text: str,
    filename: str,
    research_question: str,
    session_id: str
) -> dict:
    """Extract structured card from a single paper."""

    # Truncate to ~60k chars to stay within context
    # Use first 40k + last 20k to capture abstract, methods, and results
    if len(paper_text) > 60000:
        paper_text = paper_text[:40000] + "\n\n[...middle truncated...]\n\n" + paper_text[-20000:]

    prompt = EXTRACTION_PROMPT.format(
        research_question=research_question,
        paper_text=paper_text
    )

    message = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=1500,
        messages=[{"role": "user", "content": prompt}]
    )

    raw = message.content[0].text.strip()

    # Strip markdown code fences if Claude adds them despite instructions
    if raw.startswith("```"):
        raw = raw.split("\n", 1)[1].rsplit("```", 1)[0].strip()

    try:
        card = json.loads(raw)
        card["filename"] = filename
        card["session_id"] = session_id

        # Persist card to disk
        session_dir = CARDS_DIR / session_id
        session_dir.mkdir(exist_ok=True)
        card_path = session_dir / f"{filename}.json"
        card_path.write_text(json.dumps(card, indent=2))

        return card

    except json.JSONDecodeError:
        # Return minimal card on parse failure — don't crash the pipeline
        return {
            "title": filename,
            "filename": filename,
            "session_id": session_id,
            "methodology": "Extraction failed — open paper manually",
            "relevance_score": None,
            "error": True
        }


def get_session_cards(session_id: str) -> list[dict]:
    """Load all cards for a session."""
    session_dir = CARDS_DIR / session_id
    if not session_dir.exists():
        return []
    cards = []
    for card_file in sorted(session_dir.glob("*.json")):
        try:
            cards.append(json.loads(card_file.read_text()))
        except:
            pass
    return sorted(cards, key=lambda c: -(c.get("relevance_score") or 0))
Why truncate 40k front + 20k back? Academic papers follow a predictable structure — the abstract, introduction, and related work are at the front (where methodology context lives). Results and conclusions are at the back. The middle is usually dense implementation details that don't help with structured extraction.

PDF Ingestion

Unchanged from v1 — extract full text with page metadata, chunk for embedding.

# core/ingest.py
import fitz  # PyMuPDF
from pathlib import Path


def extract_text(pdf_path: str) -> tuple[str, list[dict]]:
    """
    Returns:
      full_text: concatenated text (for card extraction)
      pages: list of page dicts (for chunking/embedding)
    """
    doc = fitz.open(pdf_path)
    pages = []
    full_text_parts = []

    for page_num, page in enumerate(doc):
        text = page.get_text("text").strip()
        if text:
            pages.append({
                "text": text,
                "page": page_num + 1,
                "source": Path(pdf_path).name
            })
            full_text_parts.append(text)

    doc.close()
    return "\n\n".join(full_text_parts), pages


def chunk_pages(pages: list[dict], chunk_size=800, overlap=100) -> list[dict]:
    chunks = []
    for page in pages:
        words = page["text"].split()
        for i in range(0, len(words), chunk_size - overlap):
            chunk_words = words[i:i + chunk_size]
            if len(chunk_words) < 50:
                continue
            chunks.append({
                "text": " ".join(chunk_words),
                "page": page["page"],
                "source": page["source"],
                "chunk_id": f"{page['source']}_p{page['page']}_c{i}"
            })
    return chunks

Embeddings + RAG

Unchanged from v1. ChromaDB + sentence-transformers for local, free, zero-infra vector search.

# core/embeddings.py
import chromadb
from chromadb.utils import embedding_functions

ef = embedding_functions.SentenceTransformerEmbeddingFunction(
    model_name="all-MiniLM-L6-v2"
)

def get_collection(session_id: str):
    client = chromadb.PersistentClient(path=f"./chroma_db/{session_id}")
    return client.get_or_create_collection(
        name="papers",
        embedding_function=ef
    )

def index_chunks(chunks: list[dict], session_id: str) -> int:
    collection = get_collection(session_id)
    collection.add(
        documents=[c["text"] for c in chunks],
        metadatas=[{"source": c["source"], "page": c["page"]} for c in chunks],
        ids=[c["chunk_id"] for c in chunks]
    )
    return len(chunks)

def retrieve(query: str, session_id: str, n_results=15) -> list[dict]:
    collection = get_collection(session_id)
    results = collection.query(query_texts=[query], n_results=n_results)
    return [
        {"text": doc, "source": meta["source"], "page": meta["page"]}
        for doc, meta in zip(results["documents"][0], results["metadatas"][0])
    ]

Brief Synthesis

Unchanged from v1. Strict grounding prompt — every claim must cite a source chunk.

# core/synthesize.py
import anthropic, os
from core.embeddings import retrieve

client = anthropic.Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

SYNTHESIS_PROMPT = """
You are a research synthesis assistant. Produce a structured research brief
based ONLY on the provided source chunks. Every claim MUST cite its source.

STRICT RULES:
1. Cite every claim: [Filename, p.X]
2. If a question cannot be answered from the chunks, say so explicitly
3. Do NOT use knowledge outside the provided chunks
4. Do NOT invent citations

RESEARCH QUESTION: {research_question}

SOURCE CHUNKS:
{chunks_formatted}

Output sections: Synthesis | Key Debates & Contradictions | Research Gaps | Suggested Outline
"""

def generate_brief(research_question: str, session_id: str) -> dict:
    chunks = retrieve(research_question, session_id)
    if not chunks:
        return {"error": "No relevant content found in uploaded papers."}

    formatted = "\n---\n".join([
        f"[CHUNK {i+1} | {c['source']} | p.{c['page']}]\n{c['text']}"
        for i, c in enumerate(chunks)
    ])

    message = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=4000,
        messages=[{"role": "user", "content": SYNTHESIS_PROMPT.format(
            research_question=research_question,
            chunks_formatted=formatted
        )}]
    )

    return {
        "brief": message.content[0].text,
        "sources_used": list(set(c["source"] for c in chunks)),
        "chunks_retrieved": len(chunks),
        "research_question": research_question
    }

FastAPI Routes

Updated to handle both pipelines. The upload endpoint now runs card extraction and chunk indexing in parallel per file.

# main.py
from fastapi import FastAPI, UploadFile, File, Form, HTTPException
from fastapi.staticfiles import StaticFiles
from fastapi.responses import HTMLResponse, JSONResponse
import asyncio, uuid, shutil
from pathlib import Path
from core.ingest import extract_text, chunk_pages
from core.embeddings import index_chunks
from core.extract import extract_paper_card, get_session_cards
from core.synthesize import generate_brief

app = FastAPI(title="Primer")
app.mount("/static", StaticFiles(directory="static"), name="static")
UPLOAD_DIR = Path("uploads")
UPLOAD_DIR.mkdir(exist_ok=True)


@app.get("/", response_class=HTMLResponse)
async def root():
    return open("static/index.html").read()


@app.post("/api/upload")
async def upload_papers(
    files: list[UploadFile] = File(...),
    research_question: str = Form(...),   # ← now required at upload time
    session_id: str = Form(None)
):
    if not session_id:
        session_id = str(uuid.uuid4())

    session_dir = UPLOAD_DIR / session_id
    session_dir.mkdir(exist_ok=True)

    results = []

    for file in files:
        if not file.filename.endswith(".pdf"):
            continue

        # Save to disk
        file_path = session_dir / file.filename
        with open(file_path, "wb") as f:
            shutil.copyfileobj(file.file, f)

        # Extract text (used by BOTH pipelines)
        full_text, pages = extract_text(str(file_path))

        # Pipeline A: structured card extraction (Claude call per paper)
        card = extract_paper_card(
            paper_text=full_text,
            filename=file.filename,
            research_question=research_question,
            session_id=session_id
        )

        # Pipeline B: chunk + index for RAG
        chunks = chunk_pages(pages)
        n_chunks = index_chunks(chunks, session_id)

        results.append({
            "filename": file.filename,
            "chunks_indexed": n_chunks,
            "card_extracted": not card.get("error"),
            "relevance_score": card.get("relevance_score"),
            "title": card.get("title", file.filename)
        })

    return {
        "session_id": session_id,
        "papers": results,
        "total_papers": len(results)
    }


@app.get("/api/cards/{session_id}")
async def get_cards(session_id: str):
    """Return all extracted paper cards for a session, sorted by relevance."""
    cards = get_session_cards(session_id)
    return {"cards": cards, "count": len(cards)}


@app.post("/api/synthesize")
async def synthesize(
    session_id: str = Form(...),
    research_question: str = Form(...)
):
    """Generate grounded research brief from indexed chunks."""
    if not research_question.strip():
        raise HTTPException(400, "Research question required.")
    result = generate_brief(research_question, session_id)
    return JSONResponse(result)

Frontend — Paper Organizer + Brief UI

The UI now has three stages: upload → card table → brief generation. The table is sortable by relevance score and mimics your spreadsheet layout.

<!-- static/index.html — key JS logic -->

// After upload, render paper cards as a sortable table
function renderCards(cards) {
  const rows = cards.map(card => `
    <tr class="tier-${card.tier || 3}">
      <td><strong>${card.title || card.filename}</strong></td>
      <td>${card.venue || '—'}</td>
      <td>${card.year || '—'}</td>
      <td>${card.task || '—'}</td>
      <td>${card.methodology || '—'}</td>
      <td>${card.results || '—'}</td>
      <td>${(card.datasets || []).join(', ') || '—'}</td>
      <td class="score score-${card.relevance_score}">${card.relevance_score || '?'}/5</td>
      <td class="note">${card.synthesis_note || '—'}</td>
    </tr>
  `).join('');

  document.getElementById('card-table').innerHTML = `
    <table>
      <thead>
        <tr>
          <th>Title</th><th>Venue</th><th>Task</th>
          <th>Methodology</th><th>Results</th>
          <th>Datasets</th><th>Relevance</th><th>Note</th>
        </tr>
      </thead>
      <tbody>${rows}</tbody>
    </table>
  `;
}

// Export cards as CSV — matches your spreadsheet format exactly
function exportCSV(cards) {
  const headers = ['Title','Venue','Year','Task','Modality','Methodology',
    'Results','Datasets','Pretraining','Code Available','Relevance','Tier','Notes'];
  const rows = cards.map(c => [
    c.title, c.venue, c.year, c.task, c.modality, c.methodology,
    c.results, (c.datasets||[]).join('; '), c.pretraining,
    c.code_available ? 'Yes' : 'No', c.relevance_score, c.tier, c.synthesis_note
  ].map(v => `"${(v||'').toString().replace(/"/g,'""')}"`).join(','));

  const csv = [headers.join(','), ...rows].join('\n');
  const blob = new Blob([csv], {type: 'text/csv'});
  const a = document.createElement('a');
  a.href = URL.createObjectURL(blob);
  a.download = 'primer-paper-cards.csv';
  a.click();
}
The CSV export button is the killer feature moment. A researcher uploads 20 papers, gets a populated table in ~2 minutes, clicks export, and opens a perfectly formatted spreadsheet that would have taken them 4 hours to fill manually. That's the overdeliver. That's what gets shared.

Test Protocol — Day 5

Before the URL goes to anyone, run these tests in order:

Card Extraction Tests

  • Upload YOLOPX paper → verify methodology, results, and datasets match your manual spreadsheet entry
  • Upload a paper with no GitHub link → confirm code_available: false, not hallucinated URL
  • Upload a paper tangentially related to your question → confirm relevance score is 2 or 3, not 5
  • Upload a paper entirely outside your research area → confirm relevance score is 1
  • Export CSV → open in Excel/Sheets → confirm column alignment matches your spreadsheet format

Synthesis Grounding Tests

  • Ask a question directly answered in the papers → brief cites specific filenames and page numbers
  • Ask a question the papers don't address → brief says "not addressed in provided literature" rather than inventing an answer
  • Manually spot-check 3 citations → find the quoted text in the actual PDF
  • Ask the same question twice → outputs are consistent, not randomly different

Robustness Tests

  • Upload 15 PDFs at once → all process without timeout or memory error
  • Upload a scanned PDF with poor OCR → system handles gracefully, doesn't crash
  • Leave session overnight → cards and index still accessible next morning
If you pass all of these: The product is honest, works as advertised, and ready for real researchers. Put the URL in the waitlist welcome email.
